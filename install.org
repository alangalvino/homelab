#+TITLE: Homelab Installation Instructions

* Configure Synology NAS
- Set static IP: =192.168.50.10=
- Create shared folder: =/volume1/homelab=
  - Allowed IPs: 192.168.50.0/24
  - read/write access
  - Set Squash to "Map all users to admin"

* Install Proxmox on Mac Minis
Install the [[https://www.proxmox.com/en/downloads][latest Proxmox version]] manually on each Mac Mini:

** Proxmox pve1
- Hostname: =pve1.home.arpa=
- IP: =192.168.50.11=
- User: =root=
- Password: stored in KeepassXC

** Proxmox pve2
- Hostname: =pve2.home.arpa=
- IP: =192.168.50.12=
- User: =root=
- Password: stored in KeepassXC

** Proxmox pve3
- Hostname: =pve3.home.arpa=
- IP: =192.168.50.13=
- User: =root=
- Password: stored in KeepassXC

* Configure Proxmox
** Generate SSH key
#+begin_src shell
ssh-keygen -t rsa -b 4096 -f ~/.ssh/homelab -C "homelab@homelab.home.arpa"
#+end_src

** Copy SSH key to nodes
#+begin_src shell
ssh-copy-id -i ~/.ssh/homelab.pub root@192.168.50.11
ssh-copy-id -i ~/.ssh/homelab.pub root@192.168.50.12
ssh-copy-id -i ~/.ssh/homelab.pub root@192.168.50.13
#+end_src

** Create Proxmox cluster
*** SSH into a node
#+begin_src shell
ssh root@192.168.50.11
#+end_src

*** Create the cluster
#+begin_src shell
pvecm create proxmox
#+end_src

*** Add other nodes
#+begin_src shell
pvecm add 192.168.50.12
pvecm add 192.168.50.13
#+end_src

** Configure cluster
*** Install Ansible
Follow the [[https://docs.ansible.com/ansible/latest/installation_guide/installation_distros.html#installing-ansible-on-ubuntu][Ansible installation instructions]].

*** Run setup playbook
#+begin_src shell
cd ./ansible
ansible-playbook setup.yaml
#+end_src

*** Run update playbook
#+begin_src shell
ansible-playbook update.yaml
#+end_src

* Provision VMs and Podman Stacks
** Install Terraform
Follow the [[https://developer.hashicorp.com/terraform/install][Terraform installation instructions]].

** Initialize Terraform
If you are in =ansible/=, return to the project root first.
#+begin_src shell
cd ./terraform
terraform init
#+end_src

** Apply Terraform
Preview changes, then apply.
#+begin_src shell
terraform plan
terraform apply
#+end_src

Terraform provisions:
- Home Assistant VM
- Debian container repository VM (=192.168.50.100=)
- Podman and podman-compose on Debian
- systemd units that auto-start compose stacks on boot
- Stacks in =./stacks= (Pi-hole, Deye Dummycloud, Homepage)

* Stack Management
- Stack definitions live in =./stacks=
  - =stacks/pihole/compose.yaml=
  - =stacks/deye-dummycloud/compose.yaml=
  - =stacks/homepage/compose.yaml=
- Edit stack files and rerun =terraform apply= to sync and restart services.
- Service units created on Debian VM:
  - =podman-stack-pihole.service=
  - =podman-stack-deye-dummycloud.service=
  - =podman-stack-homepage.service=
